【输出格式】
输出必须是一个 JSON 数组，数组中每个元素对应一道判断+理由题。
每个元素包含以下字段：情境标签、情境、判断问题、判断结论、推理要点。

示例（仅展示结构，实际内容须按维度要求生成）：
```json
[
  {
    "情境标签": "不超过八个字的情境概括",
    "情境": "150-350字叙事性文本，包含至少一处言语信号与行为/非言语信号不一致的场景...",
    "判断问题": "[主体]在[时刻]所表达的[信念/判断]，是否与其实际持有的信念一致？请判断：一致 / 不一致，并说明你的推理依据。",
    "判断结论": "一致 或 不一致",
    "推理要点": {
      "可观测信号描述": "情境中哪些具体言语或行为构成判断依据",
      "信号与信念落差识别": "表面表达与真实信念之间的具体差异是什么",
      "真实信念推断结论": "基于上述信号，主体实际持有的信念最可能是什么"
    }
  }
]
```

注意：
- 情境须包含至少一处言语信号与行为/非言语信号不一致的场景，使"表面表达"与"真实状态"之间存在可识别的落差。
- 每道题是数组中的一个独立元素。
- 只输出 JSON 数组，不要输出任何解释、分析或其他文字。
